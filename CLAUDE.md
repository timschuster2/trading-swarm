### Rollback Chain
git checkout -- <file>  → discard edits
git reset HEAD <file>   → unstage
git revert HEAD         → undo commit (never force-push main)
Vercel/Railway: revert commit → platform redeploys previous version automatically

### On Failure
Retry once → if fails → rollback chain → REPAIR_LOOP (Diagnose/Retry/Alternative/Rollback/MinFunc/Escalate) → only then escalate to Timothy.

### Confidence Labels (Required on All Substantive Claims)
🟢 VERIFIED — confirmed from source
🟡 OBSERVED — seen in outputs/logs
🟠 INFERRED — logical conclusion
🔴 SPECULATIVE — uncertain

### Code Quality
- Commit format: <type>(<scope>): <desc>  |  types: feat/fix/docs/style/refactor/test/chore
- Atomic commits: one feature/fix per commit (waivable in Velocity Mode — log exception)
- Pre-Edit Exploration: read target file + dependencies before any edit
- Parallel execution: batch ALL file ops and bash commands in parallel — never sequential when parallel is possible
